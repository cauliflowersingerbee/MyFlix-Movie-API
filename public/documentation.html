1. Objective

This API is a server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

2.URL Endpoints & Request Methods

Request URL Method
1. Get movies /movies GET
2. Get movie by name /movies/[Title] GET
3. Get movie by genre movies/genres/[Genre] GET
4. Get data about director /movies/directors/[Director] GET
5. User registration /users POST
6. Updates user info (username) users/[Username] PUT
7. Add to favorite /users/[Username]/movies/[MovieID] PUT
8. Remove from favorite /users/[Username]/movies/[MovieID] DELETE
9. User Deregistration users/[Username] DELETE

7. This endpoint allows users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/[Username]/movies/[MovieID] PUT none A text message showing that a movie has been added to list of favorites. For example:

8. This endpoint allows users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /users/[Username]/movies/[MovieID] DELETE none A text message showing that a movie has been removed from list of favorites. For example:

"Movie successfully removed from favorites!"
9. This endpoint allows existing users to deregister (showing only a text that a user has been removed) /users/[Username] DELETE none A text message showing that a user has been removed. For example:

"Tina Ravage was successfully deleted!"
