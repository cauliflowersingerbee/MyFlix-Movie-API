1. Objective

This API is a server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

2. API design

Business logic URL HTTP method Request body data format Response body data format
1. Return a list of ALL movies to the user /movies GET none JSON object holding data about all the movies
2. Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/[info] GET none JSON object holding data about movie info
3. Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[genres] GET none JSON object holding data about a movie genre
4. Return data about a director (bio, birth year, death year) by name /movies/[directors] GET none JSON object holding data about a movie director
5. Allow new users to register /users/[registration] POST JSON object holding data about the user to add JSON object holding data about the user that was added, including an ID
6. Allow users to update their user info (username) /users/[info] PUT JSON object holding data about the user to be added JSON object holding updated user info
7. Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /movies/[favorites] PUT none A text message showing that a movie has been added to list of favorites
8. Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /movies/[favorites] DELETE none A text message showing that a movie has been removed from list of favorites
9. Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/[registration] DELETE none A text message showing that a user email has been removed
